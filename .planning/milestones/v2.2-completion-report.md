## Lessons Learned

**What Worked:**
- Phased approach (topbar → center → sidebar → drawer → theme)
- Visual foundation first (tokens, spacing, focus ring)
- E2E tests after each phase
- Preserving desktop GUI advantages (rounded corners, shadows)
- Atomic commits for each logical change

**What Was Challenging:**
- Inline card + bottom tray linkage (complex state management)
- Keyboard shortcut conflicts (macOS Option key mapping)
- Theme flash prevention (early application before React hydration)
- E2E test environment limitations (session creation)

**Recommendations for Future Milestones:**
- Continue phased approach for complex UI changes
- Invest in visual foundation early
- Test keyboard interactions thoroughly
- Consider theme system from the start
- Improve E2E test environment to support full workflows
