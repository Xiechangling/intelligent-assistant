<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| assistant proposal → native execution command | Untrusted command and path input cross into OS process execution |
| native execution response → UI tray | Captured output and review metadata become user trust signals |
| git repo state → review surface | Absence of git context can degrade review artifacts without blocking execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Elevation of privilege | `src-tauri/src/execution_service.rs` | mitigate | Keep canonical path validation in `validate_working_directory()` and add automated escape-path tests for `execute_approved_command`. |
| T-04-05 | Information disclosure | `src/app/layout/BottomPanel.tsx` | mitigate | Render exact workspace and working-directory context together before execution so the user can detect mismatched target paths. |
| T-04-06 | Repudiation | degraded review path | mitigate | Emit explicit `system` output for review-unavailable scenarios and verify the tray surfaces it distinctly from a clean no-change run. |
| T-04-07 | Denial of service | hidden tray state | mitigate | Auto-expand the tray when approval or execution begins so failures/output are not effectively hidden behind collapsed UI state. |
</threat_model>
